SAP CPQ Code Sync Utility

Welcome to the SAP CPQ Code Sync tool! (now with 100% more GUI!)
This page will allow you to pull code from CPQ environments, and then push modifications to those files back to CPQ!
If you encounter any problems be sure to send in the error report when asked!
To get started, fill in your credential information, and the folder you want to put the code, below.
When everything is filled in, click Save Credentials to save your information to your PC
You can save multiple sites, production or sandbox, and choose between them in the drop-down.
When ready, click "Refresh Scripts" and wait for the process to finish.

To get code back into CPQ you must first Start Watcher
Everything you can download from CPQ can be uploaded back to CPQ, and only takes a second or so.
Once the watcher is enabled you just need to "save" changes to a file in the "folder location" you chose to watch.
You cannot download code while the watcher is watching, this is by design.

If you have any questions, please contact Eric Hoftiezer at erichoftiezer@gmail.com.

About the App


This app is a collection of tools that we hope you find useful. Our goal was to create a single, easy to use platform that allows you access information quickly and efficiently. We're constantly working to improve and update the program. If you have an application that you'd like to add to this app, please reach out to Eric Hoftiezer!

Our Stack: Electron | JavaScript | Python


We used Electron as our framework to connect the front and back ends of the program. All front end functionality has been written in JavaScript. Our goal for V2 is to re-organize the framework to include knockout JavaScript. For the backend, we used only python to create the program's primary functionality.

Our Applications


Currently, we have 2 full functioning applications, the exporter and parser. While the weather app is present, we can't take any credit considering this is the demo app by electron. The exporter will allow for all html and python scripts to be exorpted for easier use. A future release will include the code pulled from the exporter to create a development environment for easier debugging. The parse will take in code in SAP's language and parse it to be easier to read. We also want to mention an application by Paul Trozan that will also be included in the next release, which will allow for the autonomy of tedious tasks in CPQ.
